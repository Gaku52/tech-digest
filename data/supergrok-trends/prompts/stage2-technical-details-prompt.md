# 🔬 Stage 2: 技術的詳細プロンプト

## 目的
Stage 1で抽出したトレンドの技術的詳細を深掘りし、充実した記事内容を構築する。

## 前提条件
- Stage 1の`stage1-trends.json`が完成していること
- 各トレンドの`technicalKeywords`と`sources`を使用

## 実行指示

```
あなたは日本語技術ライターです。以下のトレンドについて、技術的詳細を深掘りしてください。

【重要】
- すべての出力は**日本語**で記載してください
- 英語のツイート・ドキュメントは日本語に翻訳してください
- 技術用語は適切に日本語化してください（例: "parameters" → "パラメータ", "context length" → "コンテキスト長"）
- 固有名詞（製品名、企業名）は原語のまま使用してください

【入力情報】
{Stage 1のトレンド1件分のJSONを貼り付け}

【調査方法】
1. 元ツイート（sources）を確認
2. technicalKeywordsで関連ツイート・スレッドを検索
3. 公式発表・ドキュメントへのリンクがあれば確認
4. 技術的な議論・補足情報を収集
5. すべての情報を日本語に翻訳して出力

【出力要件】
以下のJSON形式で、技術的詳細を出力してください:

```json
{
  "trendId": "trend-001",
  "background": {
    "context": "この技術が生まれた背景・業界の課題（200-300文字）",
    "previousState": "従来の状況・既存技術の問題点（150-200文字）",
    "motivation": "なぜこの技術が必要とされたか（100-150文字）"
  },
  "technicalDetails": {
    "architecture": "技術アーキテクチャの説明（200-300文字）",
    "specifications": [
      {
        "aspect": "パラメータ数",
        "detail": "1兆総パラメータ、推論時320億アクティブで効率向上（従来比4.5倍高速化、メモリ使用量60%削減）"
      },
      {
        "aspect": "コンテキスト長",
        "detail": "256Kトークン対応、Claude 3.5の200Kを28%上回る長文処理能力"
      },
      {
        "aspect": "マルチモーダル対応",
        "detail": "テキスト・画像・動画をネイティブ統合、ゼロショットSFTで視覚推論が78.4%向上"
      }
    ],
    "implementation": {
      "technology": "使用技術・フレームワーク（100-150文字）",
      "deployment": "デプロイ方法・実行環境（100-150文字）",
      "limitations": "現時点での制約事項（100-150文字）"
    }
  },
  "benchmarks": [
    {
      "name": "HLE フルセット",
      "score": "50.2%",
      "comparison": "業界平均45.3%を10.8%上回る、SOTA達成"
    },
    {
      "name": "SWE-bench Verified",
      "score": "76.8%",
      "comparison": "Claude 4.5 Opusの73.1%を超える"
    }
  ],
  "useCases": [
    {
      "scenario": "大規模コードベースのリファクタリング",
      "benefit": "256Kコンテキストにより、プロジェクト全体を一度に処理可能",
      "example": "10万行のレガシーコードを30分で分析・提案"
    },
    {
      "scenario": "マルチモーダルデバッグ",
      "benefit": "UI/UXの問題を画面キャプチャから自動検出",
      "example": "デザインバグを視覚的に特定し、CSSコードを提案"
    }
  ],
  "additionalSources": [
    "https://...",
    "https://..."
  ]
}
```

【重要な注意事項】
1. **数値の具体性**: すべての数値に単位・比較対象を明記
   - ❌ 悪い例: "高速化された"
   - ✅ 良い例: "従来比4.5倍高速化、処理時間が120秒から27秒に短縮"

2. **技術的正確性**: 推測ではなく、元ツイート・公式情報に基づく事実のみ
   - 情報が不足している場合は "詳細未公開" と明記

3. **比較の明確化**: 競合技術・従来技術との比較を数値で示す
   - ❌ 悪い例: "GPT-4より優れている"
   - ✅ 良い例: "GPT-4の43.2%に対し、50.2%のスコアを達成（16.2%向上）"

4. **実践的なユースケース**: 具体的なシナリオ・所要時間・効果を記載

5. **specifications配列**: 最低3つ、できれば5-7つの詳細なスペックを記載

【出力形式】
- JSON形式のみ
- コメント不要
- すべてのフィールドを埋める（情報がない場合は "詳細未公開" と記載）
```

## 実行方法

Stage 1で抽出した15本のトレンドについて、**1本ずつ**このプロンプトをSuperGrokに投げてください。

Critical記事（3本）とHigh記事（3本）は特に詳細に、Medium/Lowは簡潔に。

## 出力ファイル名

`data/supergrok-trends/YYYY-MM-DD/stage2-technical/trend-XXX.json`

例:
- `data/supergrok-trends/2026-02-05/stage2-technical/trend-001.json`
- `data/supergrok-trends/2026-02-05/stage2-technical/trend-002.json`
- ...

## 次のステージへの引き継ぎ

これらのJSONファイルを使って、Stage 3でコミュニティの反応を収集します。
